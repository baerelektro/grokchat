# Пояснения к JSON-конфигам (mobile)

Этот файл нужен, потому что JSON не поддерживает комментарии по стандарту.
Чтобы не ломать инструменты и сборку, подробные пояснения вынесены сюда.

---

## 1) Файл mobile/package.json

Исходный файл: [mobile/package.json](mobile/package.json)

### Верхний уровень

- `name: "grokchat-mobile"`
  - Техническое имя npm-пакета проекта.
  - Используется в экосистеме Node/npm и tooling.

- `version: "0.1.0"`
  - Версия мобильного приложения в npm-формате.
  - На ранней стадии обычно отражает итерации MVP.

- `private: true`
  - Запрещает случайную публикацию пакета в публичный npm registry.
  - Для внутренних приложений это правильный и безопасный дефолт.

- `main: "index.js"`
  - Точка входа JS-приложения для Expo/Metro.
  - Через этот файл регистрируется корневой компонент.

### scripts

- `start: "expo start"`
  - Базовый запуск dev-сервера Expo.

- `android: "expo start --android"`
  - Запуск и попытка открыть Android-таргет.

- `ios: "expo start --ios"`
  - Запуск и попытка открыть iOS-таргет.

- `web: "expo start --web"`
  - Запуск web-режима (если нужен быстрый UI-просмотр в браузере).

### dependencies

- `expo: "~54.0.33"`
  - Базовая платформа Expo SDK.
  - `~` фиксирует minor-линию и допускает patch-обновления.

- `expo-status-bar: "~3.0.9"`
  - Управление статус-баром в Expo-приложении.

- `react: "19.1.0"`
  - Версия React, совместимая с выбранным Expo SDK.

- `react-native: "0.81.5"`
  - Базовый runtime React Native для этого SDK.

- `react-native-safe-area-context: "~5.6.0"`
  - Корректная работа отступов safe-area (чёлки, индикаторы, системные зоны).

## 2) Файл mobile/app.json

Исходный файл: [mobile/app.json](mobile/app.json)

### Блок expo

- `name: "GrokChat"`
  - Человекочитаемое имя приложения.
  - Может отображаться в Expo/на устройстве в зависимости от режима сборки.

- `slug: "grokchat-mobile"`
  - Уникальный идентификатор проекта в Expo-экосистеме.

- `version: "0.1.0"`
  - Версия приложения на уровне Expo-конфига.

- `orientation: "portrait"`
  - Принудительная портретная ориентация.
  - Для чатового MVP это упрощает layout и уменьшает количество edge-case вёрстки.

- `userInterfaceStyle: "automatic"`
  - Следование системной теме устройства (светлая/тёмная) там, где это применимо.

- `assetBundlePatterns: ["**/*"]`
  - Шаблон включения ассетов в bundle.
  - Для MVP часто используют широкий шаблон, потом сужают под оптимизацию размера.

### expo.android

- `package: "com.baerelektro.grokchat"`
  - Уникальный Android package id приложения.
  - Используется при сборке/публикации, должен оставаться стабильным.

- `softwareKeyboardLayoutMode: "resize"`
  - Экран ресайзится при открытии клавиатуры.
  - Это важно для чата, чтобы поле ввода и последние сообщения не перекрывались.

### expo.ios

- `bundleIdentifier: "com.baerelektro.grokchat"`
  - Уникальный iOS bundle identifier.
  - Аналог Android package id для iOS-экосистемы.

## 3) Почему НЕ комментируем JSON прямо в файлах

- Чистый JSON не разрешает комментарии вида `//` или `/* ... */`.
- Добавление комментариев сломает парсинг в Expo/Node-инструментах.
- Поэтому безопасный способ — отдельный поясняющий markdown рядом.
