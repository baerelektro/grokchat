# GrokChat Mobile (Expo)

Минимальный мобильный UI для вашего Rust gateway.

Этот README — быстрый практический гайд: как поднять backend, как поднять мобильный UI,
как подключиться и где искать проблему, если не коннектится.

Текущая версия: Expo SDK 54.

Важно: документ описывает именно текущий MVP-подход через gateway,
а не «полностью нативный mobile libp2p peer» (это отдельный трек в `mobile-libp2p`).

Пояснения по каждому полю JSON-конфигов смотри в отдельном файле:
- [mobile/JSON_CONFIG_COMMENTS.ru.md](mobile/JSON_CONFIG_COMMENTS.ru.md)

## Что умеет

Ниже перечислен текущий функционал без обещаний лишнего.

- подключаться к `ws://<server-ip>:8080/ws`
- показывать входящие сообщения
- отправлять сообщения в p2p-чат

Практически: мобильное приложение выступает UI-клиентом к gateway,
а p2p-распространение делает backend-нода.

## 1) Запусти Rust backend

Сначала всегда запускается backend — без него UI подключаться некуда.

В корне проекта:

1. `cargo run`
2. Убедись, что в консоли есть строка про gateway:
   `UI gateway запущен на http://0.0.0.0:8080`

Если этой строки нет, значит gateway не поднялся (ищи ошибку выше по логу).

## 2) Запусти mobile UI

Последовательность именно такая: install -> start -> открыть на устройстве.

В папке `mobile`:

1. `npm install`
2. `npx expo start`
3. Открой через Expo Go на телефоне (QR-код)

Если в PowerShell ошибка про `npx`/ExecutionPolicy, используй:

- `npx.cmd expo start`

Это частый Windows-нюанс и нормальный обходной путь.

## 3) Подключение в приложении

Поля подключения в UI:

- `IP сервера` = IP машины, где запущен Rust (например `192.168.1.10`)
- `Порт` = `8080`
- Нажми `Подключить`

Совет: не указывай `localhost`, если подключаешься с телефона —
телефон должен видеть именно IP компьютера в локальной сети.

## Формат WS

Этот блок фиксирует контракт текущего MVP-протокола между UI и gateway.

Приложение отправляет:

```json
{"type":"send_message","text":"привет"}
```

Backend присылает:

```json
{"type":"chat_message","from":"<peer_id>","text":"..."}
```

## Важно

Ключевые условия работоспособности:

Телефон и ПК должны быть в одной локальной сети.
Если не коннектится, проверь firewall на порту `8080`.

Дополнительно:
- если есть VPN на телефоне или ПК, временно выключи для теста;
- если роутер изолирует клиентов (AP Isolation), устройства могут не видеть друг друга;
- при нестабильной сети проверь, что backend продолжает держать WS без рестартов.
